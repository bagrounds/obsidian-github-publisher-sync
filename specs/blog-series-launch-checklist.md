# 🚀 Blog Series Launch Checklist

## 🎯 Overview

📋 This checklist covers everything that must be included in a PR to launch a new auto blog series. 🔧 Adding a new series requires creating a Haskell config module, registering it in the central registry, and updating several documentation files.

## ✅ PR Checklist

### 📦 Required New Files

- [ ] **`haskell/src/Automation/Series/{PascalCaseId}.hs`** — Haskell module exporting `series :: DiscoveredSeries` with seriesId, seriesName, seriesIcon, priorityUser, scheduleTime, modelChain, contextQueries, and searchGrounding fields
- [ ] **`{series-id}/AGENTS.md`** — System prompt defining the series identity, voice, editorial standards, post structure, topics, and recap guidelines. Must include YAML frontmatter with share, no_social, title, URL, Author, and tags fields
- [ ] **`{series-id}/YYYY-MM-DD-slug.md`** — Inaugural seed post written by the PR author (not auto-generated). This gives the series a first post so the automation has context for subsequent posts. Must follow the standard frontmatter format with title, URL, Author, aliases, and navigation breadcrumb
- [ ] **`specs/{series-id}.md`** — Product and engineering spec documenting the series overview, architecture, configuration, post structure, schedule, editorial standards, recaps, topics, testing, and files

### 📝 Required Code Updates

- [ ] **`haskell/src/Automation/Series.hs`** — Add a qualified import for the new module and include `{PascalCaseId}.series` in the `allSeries` list
- [ ] **`haskell/automation.cabal`** — Add `Automation.Series.{PascalCaseId}` to `exposed-modules` in the library stanza

### 📝 Required Documentation Updates

- [ ] **`README.md`** — Four updates needed:
  - Content Organization table: add row for `{series-id}/` directory
  - Scheduled Tasks table: add row with Pacific hour and description
  - Configuration Variables table: add `{SERIES_ID}_PRIORITY_USER` env var
  - Specs table: add row linking to `specs/{series-id}.md`
- [ ] **`specs/blog-generation.md`** — Two updates needed:
  - Overview paragraph: increment the series count (e.g., four to five)
  - Blog Series Configuration table: add row with series ID, icon, schedule hour, priority user, and base URL
- [ ] **`specs/scheduled-tasks.md`** — Three updates needed:
  - Data Flow diagram: add `blog-series:{series-id}` line in the task list
  - Blog Series schedule table: add row with earliest Pacific hour, task ID, and description
  - Model Fallback Chain table: add row with series name, model chain, and priority user env var. Update the grounding note if the new series uses Google Search grounding

### 🤖 Standard PR Requirements

- [ ] **`ai-blog/YYYY-MM-DD-N-slug.md`** — AI blog post describing the new series launch (per AGENTS.md PR Blogs section)

## 🔧 What the Automation Handles (Do NOT Create in PR)

🚫 The following are automatically created at runtime by `RunScheduled.hs` and should NOT be included in the PR:

- **`{series-id}/index.md`** — Generated by `ensureFileInVault` on first run using `generateSeriesIndex`
- **Content directory creation** — `createDirectoryIfMissing` handles this when the first post is written
- **Vault sync** — AGENTS.md, posts, and index are synced to the Obsidian vault automatically
- **Subsequent daily posts** — Generated by the scheduler at the configured hour

## 📐 Configuration Reference

📋 See [Blog Series Configuration spec](blog-series-discovery.md) for the full Haskell module format, convention-based derivation rules, and context query examples.

### 📝 Quick Reference: DiscoveredSeries Fields

| 🏷️ Field | 📊 Type | 📝 Description |
|---|---|---|
| `seriesId` | `Text` | Unique kebab-case identifier matching the content directory |
| `seriesName` | `Text` | Display name for the series |
| `seriesIcon` | `Text` | Emoji icon for the series |
| `priorityUser` | `Maybe Text` | GitHub handle whose comments get priority flagging (`Nothing` if none) |
| `scheduleTime` | `TimeOfDay` | Pacific time — use `TimeOfDay hour 0 0` |
| `modelChain` | `NonEmpty Gemini.Model` | Primary model followed by fallbacks |
| `contextQueries` | `[ContextQuery]` | Use `defaultContextQueries seriesId` or custom list |
| `searchGrounding` | `Bool` | Whether to use Google Search grounding |

### 🌐 Model Selection Guide

- 🔍 Series that need **Google Search grounding** (current events, news) must lead with `Gemini.Gemini25Flash` because grounding is only available on the free tier for Gemini 2.5 models
- 🤖 Series that do **not** need grounding can use `Gemini.Gemini31FlashLite` or `Gemini.Gemini3Flash` for higher quality prose
- 🔄 Always include at least two models in the chain for fallback resilience

### ⏰ Schedule Hour Selection

- 📊 Check existing series hours to avoid collisions (though the scheduler handles concurrent tasks)
- 🌅 Earlier hours are better for news-oriented series (fresher reporting)
- 🔄 All blog series use at-or-after scheduling: eligible at the configured hour and all subsequent hours

## 📄 AGENTS.md Frontmatter Format

📋 The AGENTS.md file should follow this frontmatter structure:

```
---
share: true
no_social: true
title: {icon} {Series Name} — AGENTS.md
URL: https://bagrounds.org/{series-id}/AGENTS
Author: "[[{series-id}]]"
tags:
---
# {icon} {Series Name} — AGENTS.md
```

## 📄 Inaugural Post Format

📋 The seed post should follow this frontmatter structure:

```
---
share: true
aliases:
  - YYYY-MM-DD | {icon} Title {icon}
title: YYYY-MM-DD | {icon} Title {icon}
URL: https://bagrounds.org/{series-id}/YYYY-MM-DD-slug
Author: "[[{series-id}]]"
tags:
---
[[index|Home]] > [[{series-id}/index|{icon} {Series Name}]]
# YYYY-MM-DD | {icon} Title {icon}

[Post content following the series AGENTS.md guidelines]

✍️ Written by [model name]
```
